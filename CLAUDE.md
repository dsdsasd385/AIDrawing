# AI 자동차 드로잉 체험 (Unity + ComfyUI)

전시장 설치물: 관람객이 마우스로 자동차를 그리면 로컬 SD 1.5가 실사 이미지로 변환.
대화·질문·주석은 **한국어**로 한다.

## 필독 문서 (Assets/Docs/)

작업 착수 전 순서대로 읽는다:

1. `AI자동차드로잉체험_작업계획서.md` — 확정 설계. 여기 어긋나는 구현 금지
2. `AI자동차드로잉체험_작업현황.md` — 진행 상태와 다음 작업
3. `AI자동차드로잉체험_인수인계.md` — 환경 구성, 워크플로 튜닝, 함정 목록

## 문서 갱신 의무

**모든 작업 세션이 끝나면 작업현황 §5 「문서 갱신 절차」 체크리스트대로 세 문서를 갱신한다.**
요약: 현황은 매번, 계획서는 설계 변경 시(+§18 변경 이력), 인수인계는 새 함정·환경 변경 시.
같은 내용을 두 문서에 쓰지 않는다 (상태=현황, 설계=계획서, 환경·함정=인수인계).

## 핵심 경로

- 작업 씬: `Assets/Scenes/AI자동차드로잉체험.unity` (단일 씬 + 패널 상태머신)
- 코드: `Assets/Scripts/AIDrawing/` (namespace `CarDrawing.*`, 한국어 주석 — 계획서 §17 규칙)
- 워크플로: `Assets/StreamingAssets/ComfyUI/car_workflow_api.json` (노드 ID 변경 시 클라이언트도 수정)
- ComfyUI 서버: `C:\Users\HULIAC\ComfyUI`, 기동은 `Tools\run_comfyui.bat`, 주소 `http://127.0.0.1:8188`
