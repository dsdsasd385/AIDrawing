## 1. 완료된 작업

### 코어 시스템 (Assets/Scripts/Core)

| 파일 | 역할 |
| --- | --- |
| `GameManager.cs` | 최상위 매니저. 초기화 순서 관리, 체험 플로우(인트로→에피소드→클로징) 상태 전환, 미사용 시 자동 메인 복귀 |
| `DataManager.cs` | JSON 외부 데이터 로드. 우선순위: exe 옆 `Data/`·`Config/` → `StreamingAssets` → 내장 기본값. 파일 없으면 자동 생성 |
| `DefaultEpisodes.cs` | 에피소드 1~5 기본 데이터(Default Data) 내장. JSON 손상 시 자동 복구용 |
| `LogManager.cs` | 모든 Unity 로그를 `Log/log_yyyyMMdd.txt` 에 기록. 관리자 화면용 최근 로그 메모리 보관 |
| `ScoreManager.cs` | 에피소드별 독립 점수. 최고 점수 유지, PlayerPrefs 저장, 전체 초기화 |
| `AudioManager.cs` | BGM/효과음 2채널. 리소스 없으면 조용히 생략(오류 대응), Config 로 on/off |

### 데이터 구조 (Assets/Scripts/Data)

- `GameData.cs` — `StoryData`, `EpisodeData`, `BlockEntry`, `ParamEntry`, `ConfigData` 직렬화 클래스
- 실행 시 자동 생성되는 외부 파일: `Data/Story.json`, `Data/Episode1~5.json`, `Config/Config.json`
- 점수 기준·블록 효과 값·문구가 모두 JSON Params 로 노출되어 운영자가 수정 가능 (작업계획서 14~18장)

### 블록 시스템 (Assets/Scripts/Blocks)

- `BlockInstance.cs` — 조립된 블록의 런타임 모델 + 종류별 색/모양 기호 스타일 (`ㄱ ㅡ ㄴ ㄷ U ƒ`)
- `BlockAssemblyPanel.cs` — [시작]~[종료] 고정, 삽입 위치 자동 스냅, 순서 교체, 밖으로 드래그 시 삭제, 최대 8개 제한
- `BlockView.cs` / `PaletteBlockView.cs` — 조립 블록·팔레트 견본의 드래그 처리 (마우스/터치 공용)
- `DragGhost.cs` — 드래그 중 반투명 잔상 + 삽입 위치 표시선

### 공통 UI (Assets/Scripts/UI)

- `UIFactory.cs` — 코드 기반 UI 생성 (리소스 미제작 대응). 한글 폰트(맑은 고딕 동적 폰트) 처리
- `UIManager.cs` — 작업계획서 4장의 5분할 레이아웃(상단 제목/왼쪽 팔레트/가운데 조립/오른쪽 결과/하단 버튼) + 팝업
- `StoryScreen.cs` — 인트로/클로징 화면, 등장 애니메이션(지구 확대+문구 페이드), Story.json 문구 표시

### 에피소드 프레임 (Assets/Scripts/Episodes)

- `EpisodeBase.cs` — 공통 흐름: 팔레트 구성 → 블록 검사 → 순차 실행 → 점수 계산 → 결과 팝업
- `EpisodeManager.cs` — 에피소드 전환(다음/스킵/재진입), 하단 버튼(초기화/힌트/실행) 처리

### Episode 1 「지구 온도 낮추기」 — `Episode1Controller.cs`

- 학습 요소: **변수** — 블록마다 `늘리기/줄이기` 드롭다운 선택
- 블록 4종: 화력 발전(+20) / 전기 자동차(-10) / 공원(-15) / 미세먼지(+10) — 값은 JSON 수정 가능
- 실행 결과: CO₂ 게이지·수치, 지구 온도 환산, 지구 색상 변화(파랑↔빨강) 실시간 애니메이션
- 점수: 최종 CO₂ 기준 0~3점 (기준값 `Episode1.json` Params: Score3Co2/Score2Co2/Score1Co2)
- 성공 판정: 2점 이상

### Episode 2 「빙하 지키기」 — `Episode2Controller.cs`

- 학습 요소: **명령 순서(순차 실행)** — 5개 명령을 올바른 순서로 조립
- 블록 5종: 빙하 두께 측정 → 빙하 크기 측정 → 바닷물 온도 측정 → 샘플 모으기 → 북극곰 지키기 (정답 순서는 JSON `Extra` 값으로 관리)
- 실행 결과: 명령별 순차 애니메이션(빙하 반짝임), 잘못된 순서마다 빙하 크기 감소(-15%)·바닷물 온도 상승, 빙하 40% 이하 시 북극곰 위험 표시
- 점수: 정위치 블록 개수 기준 1~3점 (`Score3Correct`/`Score2Correct` Params), 만점만 성공 판정
- 검사 규칙: 명령 2개 미만이면 실행 거부 (안내 팝업)

### Episode 3 「자연재해 대응하기」 — `Episode3Controller.cs`

- 학습 요소: **조건문** — 「만약 [재난]이라면 [대응]」 블록에 재난/대응 드롭다운 2개
- 재난 6종(폭염/지진/태풍/산사태/해일/산불) × 대응 4종(물 공급/대피 안내/시설물 관리/소방 드론)
- 정답 매핑·재난 발생 횟수(EventCount=3)·점수 기준 모두 `Episode3.json` Params 로 관리 (`Answer_폭염` 등)
- 실행 결과: 랜덤 재난 발생(중복 없음) → 조건 블록 매칭 판정 → 구조 로봇 대응 → 마을 색상 변화 연출
- 점수: 구조 성공 횟수 기준 1 / 3 / 5점, 3점 이상 성공 판정
- 검사 규칙: 같은 재난의 조건 블록 중복 시 실행 거부 (안내 팝업)

### Episode 4 「탄소 배출 줄이기」 — `Episode4Controller.cs`

- 학습 요소: **반복문** — 행동 블록마다 반복 횟수(1~5회) 드롭다운, 같은 행동이 한 회씩 반복 실행되는 과정을 시각화
- 블록 8종: 좋은 행동 4(걷기/재활용/플러그 뽑기/물 절약, 탄소 감소) + 나쁜 행동 4(자동차/일회용품/과다 전기/과다 물, 탄소 증가)
- 실행 결과: 반복 1회마다 탄소 발자국 수치·크기·색상 변화 (많을수록 크고 어두움), 게이지 연동
- 점수: 탄소 감소량 ÷ `ScorePerReduce`(기본 5) 점, 0점 이상 상한 없음. 탄소가 시작보다 줄면 성공
- 시작 수치(StartCarbon=30)·최대 반복(MaxRepeat=5)·행동별 변화량 모두 `Episode4.json` 으로 관리

### Episode 5 「지구 회복하기」 — `Episode5Controller.cs`

- 학습 요소: **함수** — 단계 블록 4개(탄소 줄이기→신재생에너지→쓰레기 줄이기→계산하기)로 '지구 회복 프로젝트' 함수를 정의하고, 호출 블록으로 실행
- 호출 블록 1개가 함수 전체 단계를 순차 실행하는 연출로 '함수 재사용' 개념을 시각화
- 실행 결과: 호출마다 병든 회색 지구가 초록으로 회복, 회복 게이지 상승
- 점수: 호출 1회당 (정순서 단계 수 × StepHeal=2) 회복, 총 회복량 = 점수 (1~50점, 호출 최대 6회 제한)
- 성공 판정: 함수 4단계를 모두 정순서로 완성했을 때
- 검사 규칙: 단계 블록 없음 / 호출 블록 없음 각각 별도 안내 팝업
- `EpisodeStubs.cs` 임시 파일 삭제 완료 → 5개 에피소드 전부 정식 구현

### 전시장 운영 기능 (Assets/Scripts/UI)

- `AdminPanel.cs` — 관리자 모드 (작업계획서 9장, 22장)
  - 진입: **F12 키** 또는 **화면 왼쪽 아래 구석(100×100px) 3초 내 5회 터치** → 숫자 키패드 비밀번호 입력 (기본 `0000`, Config.json 수정 가능)
  - 기능: 에피소드 선택(1~5 바로 이동) / JSON 다시 읽기(현재 에피소드 자동 재시작) / 점수 확인 / 로그 확인(최근 200건 스크롤) / 사운드 테스트 / 애니메이션 테스트 / 전체 점수 초기화 / 전체 초기화 / 버전 표시
- `ScreenSaver.cs` — 화면 보호 모드 (작업계획서 21장)
  - 인트로 화면에서 IdleTimeoutSec(기본 180초) 추가 방치 시 검은 화면 + 깜빡이는 안내 문구
  - 아무 곳이나 터치하면 즉시 해제
- 자동 메인 복귀 — 체험 중 미입력 시 인트로로 자동 복귀 (`GameManager`)

### 씬/빌드 구성

- `Assets/Scenes/Main.unity` — `GameManager` 오브젝트 1개만 존재. 나머지 UI 는 전부 실행 시 코드로 생성
- Build Settings 에 Main 씬 등록 완료 (빌드 시 이 씬만 포함)

### 외부 데이터 (실행 시 자동 생성 확인됨)

## 2. 테스트 결과 (플레이 모드 실측)

- ✅ 인트로 화면: 한글 문구, 등장 애니메이션, 시작하기 버튼 정상
- ✅ Episode 1 진입: 제목/설명 JSON 반영, 팔레트 4종 블록 표시
- ✅ Drag & Drop: 팔레트 → 조립 영역 스냅, 시작/종료 사이 삽입 정상
- ✅ 변수 드롭다운: 블록별 늘리기/줄이기 선택 동작
- ✅ 실행: 블록 순차 실행, CO₂ 85 계산 정확(100-20-15+10+10), 게이지/지구색 반영
- ✅ 점수 판정: 1점 → "실패" 상태 표시 및 결과 팝업 정상
- ✅ Episode 2 정순서 실행: 빙하 100% 유지, 3점/성공 판정
- ✅ Episode 2 역순 실행: 빙하 55%로 감소·온도 상승·북극곰 축소, 실행 중 버튼 잠금 확인
- ✅ Episode 3 실행: 재난 3회 랜덤 발생 → 조건 매칭 → 구조 성공 3/3, 5점/성공 판정
- ✅ 전시장 자동 복귀: 180초 미입력 시 인트로 화면 자동 복귀 실측 확인
- ✅ Episode 4 실행: 좋은 행동 4종×5회 반복 → 탄소 30→0, 6점/성공 판정, 반복 1회 단위 애니메이션 확인
- ✅ Episode 5 실행: 함수 4단계 정의 + 호출 3회 → 회복 24, 24점/성공 판정, 에피소드 누적 총점 집계 확인
- ✅ 관리자 모드: 비밀번호(0000) 키패드 진입 → 메뉴 표시 → Ep 2 바로 이동 동작 확인
- ✅ 클로징 플로우: Episode 5 → 다음 → 클로징 화면(종료 문구 + 총점 39점 표시)
- ✅ 전체 초기화: 클로징 '처음으로' → 점수 0점 초기화 + 인트로 복귀
- ✅ 외부 데이터: 첫 실행 시 Data/Config 폴더 JSON 자동 생성, Log 폴더 일자별 로그 기록 확인
